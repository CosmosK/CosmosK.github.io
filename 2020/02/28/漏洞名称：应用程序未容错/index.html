漏洞名称：应用程序未容错

Posted on 2020-02-28 In Web安全 Words： 664 Reading time ≈ 1 分钟
当系统发现异常访问或出故障时，统一出提示，应避免敏感信息泄露及显示详细错误信息。

严重性：Medium

漏洞类型：不安全的配置

安全要求：当系统发现异常访问或出故障时，统一出提示，应避免敏感信息泄露及显示详细错误信息。

0x00 简介

应用程序容错就是没有屏蔽执行过程中的错误信息，直接抛出了异常，导致一些敏感信息的泄露以及显示了详细的错误信息。

0x01 原理

一般时web应用程序接收用户输入的信息后，如果出现异常，类似于数据类型的错误、空值、非法字符等造成程序不能继续执行，导致抛出异常错误信息。

0x02 常见场景

  • 查询功能
  • 上传文件功能
  • 访问不存在的页面

0x03 测试方法

向Web提交一些恶意字符：

  • %df

  • %27

  • %20

  • ‘

  • }

  • ;

  • )

  • -

  • ../../

  • ……
