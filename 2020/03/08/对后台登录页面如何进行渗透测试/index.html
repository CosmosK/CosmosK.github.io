对后台登录页面如何进行渗透测试

Posted on 2020-03-08 In 渗透测试 Words： 3.2k Reading time ≈ 3 分钟
针对于某一个固定目标站点，如何进行相应的渗透测试，会从哪些方面入手……

0x01 检测登录功能

是否存在密码安全策略，账号锁定策略，有没有验证码功能等，若以上均没有则可以直接利用爆破。

0x02 登录密码爆破

对于登录密码的爆破可以选择自己写脚本，也可以直接使用Burp suite中Intruder模块中的Payload Processing功能，选择相应的加密方法。也可以选择固定密码，字典爆破用户名。

0x03 目录扫描

可以选择御剑工具扫，也可以使用kali系统中的fuzz,一层一层的fuzz。学习连接1 学习连接2

有些高手可以在扫除的目录中，在上传一些webshell

0x04 漏洞扫描

对于现在一些CMS已经非常成熟了，然后漏洞很难找到，但是可以从逻辑漏洞或者配置错误弱口令下手。对于一些Java开发的使用Structs 框架的，可以使用Structs2-Scan

0x05 弱口令

中国弱口令使用排行

网络设备，基本像交换机、路由器、安全设备之类的，可以尝试一下默认密码

0x06 社工

根据信息生成密码字典

kali中有cupp工具：https://github.com/Mebus/cupp

爬行网站关键信息创建字典：cewl

其他的有hashcat

0x07 逻辑漏洞

  1. 根据相应错误信息遍历用户是否存在
  2. 任意用户注册
  3. 修改注册发送包进行覆盖注册
  4. 任意密码重置
  5. 短信轰炸，（尝试在数据包后面添加空格或特殊字符要过限制）
  6. 越权
  7. 抓返回包，修改返回包参数，可能能够未授权就进入后台。Burp suite的Do intercept->Response to this request,，然后观察返回的参数，更改参数查看是否可以成功登录后台。

0x08 禁用JS

如果能够在不登录的情况下进入到应用界面，然后可以这时禁用JS，有些网页的URL跳转是由前端控制，禁用JS可能能访问部门模块，然后长传文件拿到getshell.

火狐的禁用JS插件：Disable javaScript

0x09 查看前端源代码

工具: JSFinder

在网站的js文件中提取URL，子域名的工具，用在后台登陆界面抓取一些敏感的js文件效果也很不错
